POR QUÉ CAMBIAMOS LAS CONTRASEÑAS:
===================================

El navegador (Chrome, Firefox, Safari, Edge) tiene un sistema de validación
de seguridad que detecta contraseñas débiles o que hayan sido comprometidas
en brechas de seguridad pública. 

Las contraseña "1234" es:
❌ Débil
❌ Fácil de adivinar
❌ Está en bases de datos de brechas públicas
❌ Causa advertencias de seguridad en navegadores modernos

Las nuevas contraseñas son:
✅ Fuertes (mayúsculas + minúsculas + números + caracteres especiales)
✅ Únicas para cada rol
✅ No están en bases de datos públicas comprometidas
✅ No generan advertencias de seguridad

PRÓXIMOS PASOS (RECOMENDADO):
=============================

En un entorno de producción, se debe:

1. Usar una base de datos real (MySQL/PostgreSQL)
2. Hashear las contraseñas con bcrypt o similar
3. Implementar autenticación de dos factores (2FA)
4. Usar HTTPS en lugar de HTTP
5. Implementar rate limiting para prevenir fuerza bruta
6. Usar variables de entorno para credenciales sensibles
7. Implementar logs de auditoría completos
